# Fraud Detection & Risk Analytics Dashboard

## Overview
This project focuses on analyzing financial transaction data to identify suspicious transaction patterns, fraud trends, and high-risk customer behaviors using SQL and Power BI.

---

## Tools Used
- SQL
- SQLite
- Power BI
- CSV Dataset

---

## Project Workflow
1. Created and structured a transaction database using SQLite.
2. Performed fraud analysis using SQL queries.
3. Identified suspicious transactions, high-risk cities, payment methods, and device types.
4. Exported transaction data into CSV format.
5. Built an interactive Power BI dashboard with KPI cards, trend analysis, charts, and slicers.
6. Generated visual insights for fraud monitoring and risk analysis.

---

## Key Features
- Fraud percentage analysis
- High-risk city identification
- Fraud trend monitoring
- Payment method risk analysis
- Device-based fraud analysis
- Interactive dashboard filters and slicers
- KPI-based monitoring dashboard

---

## Dashboard KPIs
- Total Transactions
- Fraud Cases
- Fraud Percentage
- Average Fraud Amount

---

## SQL Analysis Performed
- Total transaction analysis
- Fraud percentage calculation
- High-value transaction detection
- Fraud by city analysis
- Fraud by payment method analysis
- Fraud by device type analysis
- Monthly fraud trend analysis
- Merchant category fraud analysis

---

## Dashboard Visualizations
- KPI Cards
- Fraud by City (Bar Chart)
- Fraud by Payment Method (Pie Chart)
- Fraud Trend Over Time (Line Chart)
- Fraud by Device Type (Donut Chart)
- Interactive Slicers

---

## Key Insights
- High-value transactions showed increased fraud probability.
- Credit card transactions had higher fraud occurrence.
- Fraud activity varied across cities and device types.
- Interactive dashboards improved fraud visibility and monitoring.
- Trend analysis helped identify periods with suspicious transaction activity.
