# Banking-Fraud-Analysis
# Fraud Detection & Risk Analytics

## Project Overview
This project focuses on analyzing financial transaction data to identify suspicious transaction patterns, high-risk customer behavior, and potential fraud activities using SQL and Excel.

---

## Objectives
- Detect abnormal transactions
- Identify high-frequency transaction patterns
- Analyze risky geographic regions
- Monitor fraud trends
- Build analytical dashboards

---

## Tools Used
- SQL
- Excel
- CSV Dataset

---

## Key Analysis Performed
- Fraud percentage analysis
- High-value transaction detection
- Geographic fraud analysis
- Fraud by payment method
- Spending spike detection
- Monthly fraud trends

---

## SQL Concepts Used
- GROUP BY
- HAVING
- Aggregate Functions
- CASE WHEN
- Filtering
- Subqueries

---

## Dashboard KPIs
- Total Transactions
- Fraud Cases
- Fraud Percentage
- Average Fraud Amount
- High-Risk Cities
- Payment Method Analysis

---

## Key Insights
- High-value transactions showed increased fraud probability.
- Online transactions contributed to most fraud cases.
- Certain cities showed higher fraud concentration.
- Credit card transactions had elevated fraud frequency.

---

## Project Outcome
The project improved fraud visibility through SQL-based analysis and dashboard reporting, helping identify suspicious transaction behavior and risky customer patterns.
